# How the Fixture Is Structured

The importer discovers parts from folders with numeric prefixes. In this repository, `01-getting-started` becomes the first part.

Inside each part, chapter files are also ordered by numeric prefix.

## Repository Layout

```text
example-markdown-book/
  README.md
  01-getting-started/
    01-welcome.md
    02-structure.md
  02-going-deeper/
    01-markdown-features.md
    02-wrap-up.md
```

The next chapter checks a few Markdown features: [Markdown Features](../02-going-deeper/01-markdown-features.md).
